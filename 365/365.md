Mantap Mas Agis! Tabel Checklist yang kamu buat sudah sangat standar dan profesional untuk level Enterprise seperti KPK.

Ini panduan **"Klik-per-Klik" (Prakteknya)** supaya kamu bisa isi kolom **Status (OK/Not OK)** itu dengan data valid.

Buka **Microsoft 365 Admin Center** (`admin.microsoft.com`) dan **Exchange Admin Center**, lalu ikuti langkah ini:

---

### **A. Keamanan & Akses (Security & Identity)**

**1. Audit MFA (Multi-Factor Authentication)**

* **Caranya:**
* Masuk ke **Users** > **Active users**.
* Klik menu **Multi-factor authentication** di bagian atas (kadang ada di dalam menu titik tiga `...`).
* Lihat status di samping nama user (terutama Admin).


* **Status OK jika:** Statusnya *Enforced* atau *Enabled* (terutama untuk Global Admin).
* **Status Not OK jika:** Masih *Disabled*.

**2. Review Global Admin**

* **Caranya:**
* Masuk ke **Users** > **Active users**.
* Klik **Filter** (ikon corong) > pilih **Global admins**.
* Hitung jumlahnya.


* **Status OK jika:** Jumlahnya 2 sampai 4 orang.
* **Status Not OK jika:** Cuma 1 (bahaya kalau lupa password) atau lebih dari 5 (terlalu banyak, risiko keamanan).






**3. Review Sign-in Logs**

### a. Jalur Global (Yang Paling Benar untuk Laporan PM)

**👉 `Monitoring & health` > `Sign-in logs**`

* **Fungsi:** Ini adalah kamera pengawas utama. Layar ini menampilkan log masuk **KESELURUHAN** untuk **SEMUA USER** di KPK secara bersamaan.
* **Kapan dipakai:** Saat kamu melakukan Preventive Maintenance rutin seperti sekarang. Kamu bisa melihat tren global (misal: "Oh, hari ini ada 50 percobaan login dari China ke berbagai akun").
* **Kesimpulan:** Gunakan jalur ini untuk mengecek poin nomor 3 di checklist kamu.

### b. Jalur Spesifik (Untuk Investigasi Satu Orang)

**👉 `Users` > `All users` > (Pilih 1 Nama) > `Sign-in logs**`

* **Fungsi:** Ini adalah kamera pengawas khusus. Layar ini hanya menampilkan log masuk untuk **SATU USER SPESIFIK** saja.
* **Kapan dipakai:** Kalau misalnya ada laporan, "Mas Agis, tolong cek akunnya Pak Direktur dong, kayaknya ada yang aneh." Nah, kamu masuk lewat jalur ini supaya datanya tidak bercampur dengan karyawan lain.

---

### c. Karena tujuan kamu adalah audit keseluruhan perusahaan, minta Pak Badar/Pak Hadi untuk menggunakan jalur yang pertama:

1. Buka `entra.microsoft.com`.
2. Scroll menu sebelah kiri ke bawah, cari bagian **Monitoring & health**.
3. Klik **Sign-in logs**.

Data yang keluar di situ sudah merangkum seluruh aktivitas kantor.

Sudah *clear* ya bedanya Mas Agis? Mau kita lanjut bedah cara ngecek poin **Nomor 4 (External Sharing Review)** sekarang?





**4. External Sharing Review**

* **Caranya:**
* Buka **SharePoint Admin Center** (`admin.sharepoint.com`).
* Menu kiri: **Policies** > **Sharing**.
* Lihat slider *External sharing*.


* **Status OK jika:** Slider tidak di paling atas (*Anyone*). Idealnya di *New and existing guests* atau *Existing guests only*.

**5. Password Policy**

* **Caranya:**
* Di Admin Center utama: **Settings** > **Org settings**.
* Tab **Security & privacy** > klik **Password expiration policy**.


* **Status OK jika:** Tercentang dan ada setting harinya (misal 90 hari), atau jika KPK memang mau *password never expire* (sesuai kebijakan internal mereka), pastikan itu dicatat.

---

### **B. Manajemen Lisensi & User**

**6. Unassigned Licenses (Lisensi Nganggur)**

* **Caranya:**
* Menu **Billing** > **Your products**.
* Lihat kolom **Available**.


* **Status OK jika:** Angka *Available* tidak 0 (artinya masih ada stok) tapi tidak terlalu banyak (misal beli 1000 sisa 500 = pemborosan).
* **Tips:** Catat berapa sisa lisensi E3 dan E1 untuk laporan.

**7. User Cleanup (User Hantu)**

* **Caranya:**
* Menu **Reports** > **Usage**.
* Scroll ke bawah cari kartu **Active users - Microsoft 365 Services**, klik **View more**.
* Lihat kolom **Last activity date**. Cari yang tanggalnya sudah lama sekali (misal >90 hari).


* **Status OK jika:** Semua user aktif.
* **Status Not OK jika:** Banyak user yang *Last activity*-nya kosong atau tahun lalu (potensi user resign tapi akun belum dimatikan).

**8. Guest User Audit**

* **Caranya:**
* Menu **Users** > **Guest users**.


* **Status OK jika:** List user tamu dikenali. Tanyakan ke Pak Afan/IT KPK: "Pak, list tamu ini masih aktif kerjasama semua?"

---

### **C. Kapasitas & Penyimpanan**

**9. Mailbox Quota (Kapasitas Email)**

* **Caranya:**
* Buka **Exchange Admin Center** (`admin.exchange.microsoft.com`).
* Menu **Recipients** > **Mailboxes**.
* Export list-nya atau sort berdasarkan **Mailbox usage**.


* **Status OK jika:** Belum ada user yang barnya merah (usage >90%).

**10. Auto-Expanding Archive**

* **Caranya:**
* Masih di **Exchange Admin Center**. Klik nama user yang mailbox-nya besar.
* Klik tab **Others**. Lihat bagian **Mailbox archive**.


* **Status OK jika:** Statusnya *Enabled* untuk user VIP/yang emailnya penuh.

**11. SharePoint Storage**

* **Caranya:**
* Buka **SharePoint Admin Center**.
* Menu **Sites** > **Active sites**.
* Lihat kolom **Storage used**.


* **Status OK jika:** Total storage bar di pojok kanan atas dashboard belum merah.

**12. OneDrive Sync Status**

* **Caranya:**
* Ini agak *tricky* karena status error ada di laptop user masing-masing.
* Sebagai Admin, cek di **Reports** > **Usage** > **OneDrive**.
* Lihat grafik **Files**.


* **Status OK jika:** Grafiknya stabil naik (artinya sinkronisasi berjalan). Kalau datar/turun drastis, mungkin ada mass error.

---

### **D. Kesehatan Infrastruktur**

**13. Service Health Check**

* **Caranya:**
* Menu **Health** > **Service health** (yang tadi kita bahas).


* **Status OK jika:** Tidak ada *Incident* merah yang sedang aktif. (Advisory biru dianggap OK/Wajar).

**14. DNS Record Health**

* **Caranya:**
* Menu **Settings** > **Domains**.
* Klik domain utama (misal `kpk.go.id` atau domain onmicrosoft-nya).
* Klik tab **DNS records**.


* **Status OK jika:** Status domain centang hijau (*Healthy*). Tidak ada *error* pada record MX atau TXT (SPF).

**15. Message Center Review**

* **Caranya:**
* Menu **Health** > **Message center**.


* **Status OK jika:** Kamu sudah membacanya sekilas.
* **Tips:** Cari info soal "Retirement" (fitur yang mau dihapus). Info ini mahal harganya buat klien karena bikin mereka siap-siap.

---

**Saran Tambahan buat Mas Agis:**
Saat di lokasi, **jangan mengubah settingan apapun** tanpa izin tertulis/lisan yang jelas dari Pak Afan/Tim KPK. Tugas PM (Preventive Maintenance) utama adalah **Memeriksa (Check)** dan **Melaporkan (Report)**. Jika ada yang *Not OK*, catat di kolom keterangan sebagai "Rekomendasi Perbaikan".
